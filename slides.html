What is the CAC 40?

Context

  • The CAC 40 is France's benchmark stock market index — the 40 largest publicly traded companies on Euronext Paris
  • Think of it like the French equivalent of the S&P 500
  • Its composition changes quarterly as companies grow, shrink, or delist
Why does this matter?
Funds that track the CAC 40 must rebalance their holdings every quarter to stay aligned with the index. Getting this right — knowing which stocks to hold and in what quantities — is the core problem this project solves.

The Problem

Index tracking

  • Given a $1B portfolio, which 40 stocks should we hold each quarter?
  • And how many shares of each, to best track (or beat) the index?
Strategy A — Cap Weighting
Weight each stock proportional to its market capitalisation. Simple, passive, mirrors the index directly.
Strategy B — Max Sharpe Ratio
Optimise weights to maximise risk-adjusted return. More complex, potentially higher performance.

Cap Weight passive

  • Each stock's weight = its market cap ÷ total market cap of all 40
  • Larger companies get a higher weight — mirrors how the index itself is calculated
  • Simple, low-turnover, low cost to implement
  • Output: a CSV of 40 weights summing to 1.0
Example
LVMH has the largest market cap in the CAC 40 (~€300B), so it receives the highest weight (~8–10%)

Max Sharpe Ratio Portfolio optimised

  • Sharpe ratio = return above the risk-free rate ÷ volatility — a measure of return per unit of risk
  • Uses historical returns to estimate expected return and covariance (how stocks move together)
  • Runs a numerical optimiser (scipy SLSQP) to find the weights that maximise the Sharpe ratio
  • Uses Ledoit-Wolf shrinkage (scikit-learn) to produce a more stable covariance estimate — avoids overfitting on short return histories

Valuation
  • Takes the strategy weights + quarter-end prices and converts them into a real share count for a $1B notional portfolio
  • Fractional shares are floored — the remainder is held as cash
  • The quarterly allocation is stored and reused daily
Daily valuation (CI)
A GitHub Actions job runs on every push to main — it fetches live prices, multiplies by the share counts, and commits the portfolio value back to the repo as a CSV.

Automation

GitHub Actions

Quarterly — Universe Update
  • Triggers on the last few days of each quarter month (Mar / Jun / Sep / Dec)
  • Fetches new CAC 40 composition, resolves ISINs, appends new quarter to quarters.txt
  • Idempotent — safe to re-run
Daily — Portfolio Valuation
  • Triggers on every push to main
  • Downloads latest prices, computes portfolio value, auto-commits output CSVs
  • Keeps a running daily history of portfolio performance
